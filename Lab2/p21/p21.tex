\section*{Gestión de procesos:}
	
  \begin{itemize}
  	\item En Linux se están ejecutando al momento un gran número de procesos:
	\begin{itemize}
	\item procesos de sistema (kernel, daemons)
	\item procesos de usuarios
	\end{itemize}
	\item En esta sección trataremos la gestión de los procesos que se están ejecutando:
	\begin{itemize}
	\item listar procesos en ejecución
	\item detener y matar procesos
	\item controlar la prioridad de ejecución
	\end{itemize}
\subsection*{Comando ps:}
 \item Existen varias herramientas para ver los procesos en ejecución, la más importante es el comando \emph{ps} (sin opciones, ps sólo muestra los procesos lanzados desde el terminal actual y con el mismo EUID que el usuario quien lo lanzó:
\begin{lstlisting}[language=bash]
$ ps 

PID TTY          TIME CMD
 1625 pts/0    00:00:00 bash
 1784 pts/0    00:00:00 ps
\end{lstlisting}

\item ps tiene un gran número de opciones, que se pueden especificar de 3 maneras:
\begin{itemize}
\item opciones UNIX: pueden agruparse y se preceden por un guión: ps -ef 
\item opciones BSD: pueden agruparse y van sin guión: ps aux
\item opciones largas GNU: precedidas de dos guiones: ps --user tomas
\end{itemize}

\begin{itemize}

\item -e ó ax ó -Af: muestra todos los procesos
\item -u (o U o --user) usuario: muestra los procesos de un usuario
\item u: salida en formato de usuario
\item j: salida en formato job (muestra PID, PPID, etc.)
\item l ó -l: salida en formato largo
\item f: muestra un árbol con la jerarquía de procesos
\item -o (ó -o format) formato: permite definir el formato de salida
\begin{lstlisting}[language=bash]
$ ps -o ruser,pid,comm=Comando
\end{lstlisting}

\end{itemize}

   \section*{Manipulación de procesos:}
   Mediante la combinación de Ctrl-C y Ctrl-Z podemos terminar y detener un proceso.
   \begin{itemize}
   \item esas combinaciones envían señales a los procesos.
   \item el comando básico para enviar señales a un proceso es kill.
   
   \end{itemize}

	\begin{itemize}
	\item kill -l lista el conjunto de señales

	\item para ver su significado, ver \emph{man 7 signal}
	Sintaxis de \textbf{kill}\\
	\begin{lstlisting}[language=bash]
	kill [señal] PID
	\end{lstlisting}
	
	\item \emph{señal} puede indicarse mediante el número o el código:
	\begin{itemize}
	
	\item \textbf{kill -9} y \textbf{kill -KILL} son equivalentes.
	\item las señales más comunes son:
		\begin{itemize}
		\item SIGHUP (1): cuelgue del terminal o muerte del proceso controlador
		\item SIGTERM (15): mata el proceso permitiéndole terminar correctamente
		\item SIGKILL (9): mata el proceso sin permitirle terminar
		\item SIGSTOP (19): para el proceso
		\item SIGCONT(18): continúa si parado.
		\item SIGINT (2): Interrupción de teclado (Ctrl-C)
		\item SIGTSTP (20): stop de teclado (Ctrl-Z)
		\item SIGQUIT (3): salida de teclado (Ctrl-\textbackslash)
		
		\end{itemize}
		
		Algunas características de las señales:
		\item La señal que se envía por defecto es TERM (15)
			\begin{itemize}
			\item los procesos pueden ignorar esta señal y no terminar.
			\item las señales KILL (9) y STOP(19) no pueden ignorarse.
			
			\end{itemize}
		\item En bash, cuando enviamos una señal SIGHUP a un shell, este se lo reenvía a todos sus hijos.
		\item Cuando cerramos un terminal en un entorno gráfico, o abandonamos una sesión, se envía un SIGHUP a todos sus hijos
		\item La mayoría de los demonios (daemons) responden a la señal SIGHUP volviendo a leer sus ficheros de configuración:
		\begin{itemize}
		\item en vez de matar y reiniciar un demonio podemos hacer un kill -HUP para reiniciarlo
		
		\end{itemize}
	\end{itemize}
	\end{itemize}
\subsection*{Otros comandos útiles}
	\textbf{pgrep} busca una lista de procesos para localizar el PID a partir del nombre (similar a \textbf{ps | grep})
	\begin{lstlisting}[language=bash]
		 $ pgrep -u root sshd # PID del proceso sshd de root
	\end{lstlisting}
\textbf{pkill} permite enviar señales a los procesos indicándolos por nombre en vez de por PID.

	\begin{lstlisting}[language=bash]
		 $ pkill -9 proceso
	\end{lstlisting}
	\begin{itemize}
		\item \textbf{Cuidado con pkill: } si hay varios procesos con el mismo nombre los mata a todos. \\

		\item \textbf{killall} similar a pkill, pero no admite patrones en el nombre, y tiene otras opciones. \\

		\item \textbf{nohup} normalmente, cuando salimos de un login shell (logout) o cerramos una un terminal, se envía una señal SIGHUP a todos los procesos hijos.
		\begin{itemize}
		\item si lanzamos un proceso en background y salimos de la sesión el proceso muere al morir el shell desde que lo iniciamos.
		El comando \textbf{nohup} permite lanzar un comando ignorando las señales SIGHUP.
		
		\end{itemize}
 		\begin{lstlisting}[language=bash]
		$ nohup comando # la salida del comando se redirige al fichero nohup.out
	\end{lstlisting}
	 \item \textbf{exec} es otra forma de crear un proceso que no se muere al morirse el shell.
	
\item \textbf{Ejecución en background y foreground}: las órdenes lanzadas en background permiten tener el control de la consola mientras se ejecuta en segundo plano. Si queremos que el proceso se ejecute en primer plano, tenemos que hacernos valer de la orden fg.\\
\begin{lstlisting}[language=bash]
$ yes > /dev/null &  # lanzado en background
[1] 27058
$ fg 1
yes > /dev/null
\end{lstlisting}

\subsection*{Comandos \textbf{nice} y \textbf{renice}}
Cuando un proceso se ejecuta, lo hace con una cierta \emph{prioridad}.
\begin{itemize}
\item las prioridades van desde -20 (más alta) hasta 19 (más baja).
\item los procesos se ejecutan con prioridad 0 por defecto. Además, un usuario normal sólo puede asignar prioridades más bajas (números positivos), mientras que \emph{roto} puede asignar prioridades más altas (números negativos).
\end{itemize}

Sintaxis:\\
\begin{lstlisting}[language=bash]
$ nice -n <ajuste> <comando>  # la prioridad es modificada por ajuste
\end{lstlisting}

Ejemplo:\\

\begin{lstlisting}[language=bash]
$ cat script1.sh
#!/bin/bash
yes > /dev/null &
$ nice -n 10 ./script1.sh 
$ ps -o pid,pri,ni,stat,cmd
  PID PRI  NI STAT CMD
 4387  19   0 Ss   -bash
 4585  19   0 R    yes
 4613  19   0 R+   ps -o pid,pri,ni,stat,cmd
$ nice -n -1 4585 # No va a funcionar al bajar la prioridad y no ser root.
\end{lstlisting}

Con \emph{renace} podemos modificar la prioridad:\\

Sintaxis:\\
\begin{lstlisting}[language=bash]
$ renice pri [-p pid] [-g pgrp] [-u user] # la prioridad es modificada por ajuste
\end{lstlisting}

Si por ejemplo:\\
\begin{lstlisting}[language=bash]
$ ./script1.sh
[1] 4455
$ renice 10 -p 4455 # cambia por pid
$ renice 3 -u user01 # cambia para los procesos de usuario especificado.
\end{lstlisting}



\end{itemize}

\subsection*{\textbf{Análisis básico del rendimiento del sistema}}

Además del ps y el top, existen otro comandos básicos que nos pueden mostrar el estado del sistema en cuanto a uso de CPU y consumo de memoria, no obstante, veremos otros más relacionados con estos usos ( vmstat y sar )

\textbf{ultime}: muesta la hora actual, el tiempo que el sistema lleva encendido, el número de usuarios conectados y la carga media del sistema para los últimos 1, 5 y 15 minutos (lo mismo que en la primera línea de top).

\begin{lstlisting}[language=bash]
$ uptime
 11:35:34 up  1:40,  5 users,  load average: 9,80, 8,26, 5,81
\end{lstlisting}

\textbf{w}: nos da la información sobre los usuarios y sus procesos.\\
\begin{lstlisting}[language=bash]
$ w
 12:08:15 up  2:13,  5 users,  load average: 9,73, 9,58, 9,21
USUARIO  TTY      DE               LOGIN@   IDLE   JCPU   PCPU WHAT
admin    tty1                      19abr14  1:05m  2.74s  2.63s -bash
admin    pts/0    192.168.1.100    19abr14 10días  0.43s  0.43s -bash
admin    pts/1    10.1.80.169      jue19    4días  0.31s  0.31s -bash
admin    pts/2    10.1.80.169      jue19    4días  0.33s  0.33s -bash
admin    pts/5    192.168.1.100    11:27    1.00s  1:43   0.00s w
\end{lstlisting}

Tenemos:\\
\begin{itemize}
\item LOGIN\@ la hora a la que se conectó el usuario.
\item IDLE tiempo que lleva ocioso el terminal.
\item JCPU el tiempo de CPU consumido por los procesos que se ejecutan en el TTY.
\item PCPU tiempo consumido por el proceso actual (el que aparece en la columna WHAT).

\end{itemize}

\textbf{free}: muestra la cantidad de memoria libre y usada en el sistema, tanto para la memoria física como para el swap, así como los bufares usados por el kernl (similar a la cabecera de top).

Ejemplo:\\
\begin{lstlisting}[language=bash]
$ free
             total       usado       libre     compart.    búffers     almac.
Mem:        508344     201480     306864          0      29324     130460
-/+ buffers/cache:      41696     466648
Intercambio:     521212          0     521212
\end{lstlisting}
\begin{itemize}
\item Columna \emph{shared} está obsoleta.
\item Opciones: -b, -k, -m, -g memoria en bytes/Kb/Mb/Gb.
\item Opción: -t muestra una línea con el total de memoria (física + swap).
\item Opción: -s delay, muestra la memoria de forma continua, delay = segundos.

\end{itemize}

\textbf{htop}: Es una herramienta gráfica, basada en top, pero con muchas más opciones de gestión de procesos. Es como el panel de tareas de Windows. Se pueden finalizar procesos y otras tareas.
